# Hospital Pulse AI (Healthcare PS-06)
Predictive Hospital Resource & Emergency Load Intelligence System. Decision-support tool that forecasts ER admissions, ICU demand, and staff workload, then surfaces alerts and actionable recommendations.

## Architecture
- **Backend:** FastAPI, pandas/numpy, scikit-learn linear baselines + lightweight LSTM-like smoother, synthetic data generator (CSV), rule-based alerts/recommendations.
- **Frontend:** React (Vite), MUI, Recharts, Axios. Clean admin dashboard with ER/ICU charts, staff workload heat map, alerts, and AI recommendations.
- **Data:** `data/synthetic_data.csv` autogenerated on backend start if missing (hourly series with ER, ICU, staffing, weather/seasonality signals).

## Why it helps hospitals
- Predicts emergency surges (next 7 days) and ICU peak risk, highlighting surge probability.
- Estimates nurse-to-patient stress to preempt burnout.
- Generates interpretable recommendations (staff reallocation, ICU prep, shift adjustments) and early alerts.

## Run locally
1) **Backend**
```bash
cd backend
python -m venv .venv
.venv\Scripts\activate  # Windows
pip install -r requirements.txt
uvicorn backend.main:app --reload --port 8000
```

2) **Frontend**
```bash
cd frontend
npm install
npm run dev -- --host --port 5173
```
Set `VITE_API_BASE=http://localhost:8000` in a `.env` if your backend port differs.

### Environment variables
- Create `backend/env/.env` with:
```
OPENAI_API_KEY=sk-...
# Optional override (default gpt-4o-mini)
OPENAI_MODEL=gpt-4o-mini
```

## API endpoints
- `POST /predict/emergency` – hourly forecast, surge probability
- `POST /predict/icu` – ICU beds required, peak risk
- `POST /predict/staff` – staff workload risk per shift
- `GET /alerts` – early warning cards
- `GET /recommendations` – actionable optimization steps
- `GET /health` – service status

## Ethical considerations
- No patient-level identifiers; synthetic or aggregated signals only.
- Supports clinicians; does not replace medical judgment. Include disclaimer in UI/README.
